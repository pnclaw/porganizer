## feature/video-library-second-improvements — 2026-04-10

### Done
- Fixed bug where downloads that completed while porganizer was offline were incorrectly marked as `Failed` instead of `Completed`
- Root cause: when a download client is unreachable (e.g. queue poll throws a connection error), the poller returned an empty result list, causing `DownloadPollService` to treat all pending items as "not found" and increment `MissedPollCount`. After 3 such polls the items were marked `Failed`.
- Introduced `DownloadPollGroupResult` wrapping `ClientReachable` flag and `HistoryCheckFailedIds` set
- `SabnzbdPoller` and `NzbgetPoller` now surface whether the queue poll succeeded (`ClientReachable`) and which per-item history lookups threw exceptions (`HistoryCheckFailedIds`)
- `DownloadPollService` skips `MissedPollCount` updates when the client is unreachable, and skips per-item updates when the history check failed (unknown status, not absent)
- Added 7 integration tests covering: client unreachable, item completed in history, history check failure, and genuinely absent → Failed happy/sad paths
- Added `POST /api/download-logs/{id}/recheck` endpoint: resets a Failed download (Queued, MissedPollCount=0, clears error/completedAt) then immediately polls the client; returns 400 if already not Failed or if no `ClientItemId` recorded
- Added "Recheck in client" button to the download log detail dialog, visible only for Failed downloads that have a `ClientItemId`; updates the item in-place in the list after the server responds

## feature/fixing-wanted-video-sync-problem — 2026-04-06

### Done
- Added `PrdbDownloadedFromIndexerSyncService`, which runs from the 15-minute `SyncWorker` and reports completed downloads to prdb.net via the new `downloaded-from-indexers` API family
- Sync is now retryable and stateful: each `DownloadLog` stores the remote prdb.net parent id, a sync fingerprint, last attempt time, last success time, and the last sync error
- Reporting is gated on file readiness: before submitting anything to prdb.net, the app refreshes the local `DownloadLogFiles` rows from disk and skips the PRDB sync until files are actually present
- Implemented incremental filename reconciliation against prdb.net: removed local files trigger PRDB filename deletes, new files trigger PRDB filename creates, and existing files are updated in place
- Added per-file `FileSize` plus remote filename ids to `DownloadLogFiles`, so local file metadata can be matched back to prdb.net child rows across worker runs
- Mapped PRDB `indexerSource` from the local indexer URL using the currently supported providers: `drunkenslug` → `0`, `nzbfinder` → `1`; unsupported indexers are skipped with a stored sync error
- Refactored download file discovery into a reusable `DownloadLogFileSyncService`, used both by `DownloadPollService` and the new PRDB background sync
- Added automated tests covering first-time create, deferred sync when files are missing, and update reconciliation with filename add/update/delete behavior
